# ADR-0001: Chakra UI v3 as design system

## Status

Accepted

## Context

The product catalogue needs a responsive grid layout, accessible form controls (search, select, buttons), loading skeletons, and card components. Options considered:

- **Chakra UI v3** — compound components, responsive props, built-in a11y
- **Tailwind CSS** — utility-first, no component abstractions
- **Vanilla CSS / CSS Modules** — no dependencies, full control

## Decision

Use Chakra UI v3 with `defaultSystem` (no custom theme).

## Consequences

- **Positive:** Responsive grid via `SimpleGrid columns={{ base: 1, sm: 2, md: 3, lg: 4 }}` replaces manual media queries. Card, Badge, Skeleton, NativeSelect provide consistent accessible UI out of the box.
- **Positive:** Must be shared as singleton across Module Federation boundary — adding `@chakra-ui/react` and `@emotion/react` to MF shared config ensures one instance.
- **Negative:** Adds ~80KB to bundle. Acceptable for this use case since the catalogue is the primary UI.
- **Negative:** Chakra v3 API differs from v2 (e.g. `colorPalette` not `colorScheme`, `fit` not `objectFit`, `NativeSelect.Root`/`.Field` compound pattern). Required consulting source references.
