# ADR-0007: Virtual scroll for product catalogue

## Status

Accepted

## Context

The product catalogue renders up to 2000 products, currently paginated at 24 per page with Next/Previous buttons. Pagination requires many clicks to browse the full dataset and breaks visual scanning flow. Virtual scrolling renders only the visible rows, keeping DOM node count constant regardless of dataset size.

## Decision

Use `@tanstack/react-virtual` (~3 KB gzip) to virtualize the product grid by **rows**, where each row contains 1–4 cards depending on the container width (measured via `ResizeObserver`).

- Virtual scroll is the **default** behaviour (`featureFlags.virtualScroll !== false`).
- Setting `virtualScroll: false` in the host's feature flags falls back to the existing pagination path.
- The scroll container uses `height: calc(100vh - 160px)` and `overflow: auto` with an `overscan` of 2 rows for smooth scrolling.
- Scroll position resets to the top when filters change.

### Alternatives considered

- **react-virtuoso**: Fights Chakra's `SimpleGrid` layout with its own wrapper divs. Requires workarounds to achieve a responsive grid.
- **react-window**: Maintenance mode, no built-in support for responsive column counts.

## Consequences

- **Positive:** DOM stays small (~40 nodes for visible rows) even with 2000+ products — improves initial render and scroll performance.
- **Positive:** Feature flag fallback preserves the paginated path for A/B testing or if issues are discovered.
- **Positive:** Responsive column count matches Chakra's breakpoint tokens, keeping visual consistency.
- **Negative:** Adds a runtime dependency (~3 KB gzip). Acceptable given the performance gain.
- **Negative:** `estimateSize` is a fixed 296 px per row; variable-height cards would need dynamic measurement.
