# ADR-0006: Error boundary for remote module isolation

## Status

Accepted

## Context

The host loads `ProductList` from the remote via `React.lazy(() => import('products/ProductList'))`. If the remote is down or the module fails to load, the lazy import rejects and React throws during render — crashing the entire host application.

## Decision

Wrap the lazy remote component in a class-based `ErrorBoundary` (using `getDerivedStateFromError`) inside a `Suspense` boundary:

```tsx
<ErrorBoundary fallback={<RemoteError />}>
  <Suspense fallback={<LoadingSkeleton />}>
    <ProductList featureFlags={{ showRatings: true }} />
  </Suspense>
</ErrorBoundary>
```

`Suspense` handles the loading state (skeleton grid). `ErrorBoundary` catches both module load failures and runtime errors from the remote component.

## Consequences

- **Positive:** Host remains functional when remote is unavailable — shows a clear error message instead of a white screen.
- **Positive:** `Suspense` provides a skeleton loading state during the initial chunk download + MSW initialization.
- **Positive:** Error boundary catches any runtime error thrown by the remote component, not just load failures.
- **Negative:** Class-based error boundary required — React does not yet support error boundaries as function components. Minimal code (~24 lines).
