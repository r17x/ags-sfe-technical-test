# ADR-0004: Singleton shared dependencies in Module Federation

## Status

Accepted

## Context

Both host and remote use React 19, Chakra UI v3, and Emotion. Without shared configuration, Module Federation bundles separate copies into each app, causing:

1. Multiple React instances — breaks hooks (different dispatcher)
2. Multiple Emotion instances — style injection conflicts, duplicate `<style>` tags
3. Multiple Chakra instances — theme context mismatch between host provider and remote components

## Decision

Configure `react`, `react-dom`, `@chakra-ui/react`, and `@emotion/react` as singleton shared dependencies in both `vite.config.ts` files:

```ts
shared: {
  react: { singleton: true, requiredVersion: '^19.0.0' },
  'react-dom': { singleton: true, requiredVersion: '^19.0.0' },
  '@chakra-ui/react': { singleton: true },
  '@emotion/react': { singleton: true },
},
```

Also align `@module-federation/vite` to `^1.7.1` in both apps (was `^1.2.6` in apps vs `^1.7.1` at root).

## Consequences

- **Positive:** Single React instance across the federation boundary — hooks work correctly.
- **Positive:** Single Chakra/Emotion instance — `ChakraProvider` in host provides theming context to remote components seamlessly.
- **Positive:** Reduced bundle size — shared deps loaded once.
- **Negative:** Version coupling — host and remote must agree on compatible versions. Enforced by `requiredVersion` for React.
