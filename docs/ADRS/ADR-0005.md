# ADR-0005: Client-side filtering with useDeferredValue

## Status

Accepted

## Context

The API returns all 2000 products at once (no server-side pagination). Filtering, sorting, and pagination must happen client-side. The search input needs some form of debouncing to avoid re-filtering on every keystroke.

Options considered:

- **setTimeout debounce** (300ms) — traditional, explicit delay
- **useDeferredValue** (React 19) — lets React defer re-renders at low priority
- **External library** (lodash.debounce, use-debounce) — additional dependency

## Decision

Use `useDeferredValue` for the search query. Derive filtered/sorted/paginated results via `useMemo` chains. Paginate at 24 items per page. Reset page to 1 on any filter change.

```ts
const deferredQuery = useDeferredValue(query);
const filtered = useMemo(() => {
  const searched = filterBySearch(data, deferredQuery);
  const categorized = filterByCategory(searched, category);
  return sortByPrice(categorized, sort);
}, [data, deferredQuery, category, sort]);
```

## Consequences

- **Positive:** `useDeferredValue` is zero-config — React automatically prioritizes the input update (keeping typing responsive) and defers the expensive filter/sort re-render.
- **Positive:** Pure filter functions in `filters.ts` are independently testable (14 unit tests) with no React dependency.
- **Positive:** Pagination keeps DOM size to 24 cards regardless of dataset size.
- **Negative:** All 2000 items are filtered in memory on each change. At this scale (~2ms on modern hardware) this is negligible. Would not scale to 100K+ items without virtualization or server-side filtering.
